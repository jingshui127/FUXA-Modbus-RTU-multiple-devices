# Modbus RTU 多从设备测试报告

## 测试概述

本次测试验证了 Modbus RTU 多从设备共享单一 COM 端口的功能，包括所有类型寄存器的读写操作。

## 测试环境

- **操作系统**：Windows
- **串口**：COM20 (虚拟串口，与 COM21 配对)
- **波特率**：9600
- **数据位**：8
- **停止位**：1
- **校验位**：无
- **从站数量**：5 个（从站 ID: 1-5）

## 测试设备配置

在 `fuxa-devices.json` 文件中为 ModbusRTU1 设备添加了以下类型的测试变量：

### 1. Coil Status (000001-065536)
- **数据类型**：Bool
- **从站**：1-5
- **地址**：1-5
- **示例变量**：Coil_1.1, Coil_2.1, ..., Coil_5.5

### 2. Digital Inputs (100001-165536)
- **数据类型**：Bool
- **从站**：1-5
- **地址**：1-5
- **示例变量**：Input_1.1, Input_2.1, ..., Input_5.5

### 3. Input Registers (300001-365536)
- **数据类型**：Int16
- **从站**：1-5
- **地址**：1-5
- **示例变量**：t_inputreg_1.1, t_inputreg_2.1, ..., t_inputreg_5.5

### 4. Holding Registers (400001-465536)
- **数据类型**：Int16, Int32, UInt32, Float32, Float64, Int64 及其小端（LE）和 Modbus 中间小端（MLE）变体
- **从站**：1-5
- **地址**：1-5, 8, 12, 16-17, 19, 21, 25, 29, 33, 35, 37
- **示例变量**：t_test_int16_01, t_test_int32_01, ..., t_test_uint32mle_05

## 修复的问题

### 1. Coil/Input 类型变量显示为空值 ✅
**问题原因**：`res.data` 为布尔数组时，未正确提取对应索引的布尔值。
**修复方案**：在 `_readMemory` 函数中添加 `bitIndex` 边界检查，确保从 `res.data` 正确获取值。

### 2. Float64 类型显示异常 ✅
**问题原因**：Buffer 创建逻辑错误，未正确处理 Uint16Array 数据。
**修复方案**：优先使用 `res.buffer`，其次从 `res.data.buffer` 创建 Buffer，最后手动构建 Buffer。

### 3. 终端日志过多 ✅
**问题原因**：代码中使用了大量 `console.log` 输出详细信息。
**修复方案**：将日志输出改为使用 `logger.debug` 和 `logger.error`，根据日志级别输出。

### 4. 新增/删除变量后数据变为空 ✅
**问题原因**：`load` 函数未清空 `memItemsMap`，`MemoryItem` 未初始化 `rawValue`，`_updateVarsValue` 跳过 `rawValue` 为 `null` 的变量。
**修复方案**：
- 清空 `memItemsMap`
- 初始化 `rawValue` 和 `changed`
- 修改 `_updateVarsValue` 始终计算 `value`

## 测试结论

所有测试均已通过，Modbus RTU 多从设备共享单一 COM 端口功能正常，支持所有类型寄存器的读写操作，包括不同字节序的数据类型。

### 关键功能验证：
1. **多从站支持**：成功实现 5 个从站的数据隔离和并发访问
2. **串口复用**：单一 COM 端口可同时服务多个从站
3. **寄存器类型支持**：所有 4 种类型寄存器均正常工作
4. **数据类型支持**：支持多种数据类型及其字节序变体
5. **变量管理**：新增/删除变量后数据保持稳定
